Secure Enclaves Framework Documentation

Installation

Getting started with SSE framework project is simple:


./rbuilds.sh --tty --debug --dockerfile ./pipeline-slc-network-al2023.dockerfile --cmd "make_all" --network --init-c 2>&1

mkdir -vp ./eif/ ;
/usr/bin/time -v -o ./eif/make_build.log \
./rbuilds.sh --tty --debug --dockerfile ./pipeline-slc-network-al2023.dockerfile --cmd "make_all" --network --init-c \
2>&1 | tee ./eif/make_build.output

./rbuilds.sh --help

./rbuilds.sh --help-ext

./rbuilds.sh --man

Changelog and Roadmap Release Schedule

We've got an internal semantic versioning and numerology =) for releases, according to roadmap release schedule:


0.1.0 - Pipeline SLC with directories transferring in minor subversions

0.2.0 - reproducible builds for EIF images

0.3.0 - Proxies with socat/UDP/DNS, transparent proxies, and high-level protocols support over vsock, all as minor subversions

0.4.0 - web protocol for RA, with VRF proofs for attestation docs, mt-runtime, mass-production of attestation docs,
        hot cache and docs cold DB (Sled) integration, all of these as minor versions

0.5.0 - file system monitor, for automagic mass-production of attestation docs, with mt-runtime integration as well, as minor subversion

0.6.0 - cryptography stack, for buffer and content encryption

0.7.0 - integration with KMS for key storage, probably TPM module usage for local key storage as minor version update

Next is a subject to change in order of versions:

0.8.0 - enclave's init system update, rewritten in Rust (can be included as minor version sooner)

0.9.0 - run framework on QEMU VM with EIF images support.
